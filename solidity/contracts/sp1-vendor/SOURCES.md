## Vendor modifications

`SP1VerifierGroth16.sol` is byte-identical to upstream EXCEPT for one line:

```diff
- import {ISP1Verifier, ISP1VerifierWithHash} from "../ISP1Verifier.sol";
+ import {ISP1Verifier, ISP1VerifierWithHash} from "./ISP1Verifier.sol";
```

This change flattens the import path because we vendored `ISP1Verifier.sol`
into the same directory rather than reproducing upstream's `contracts/src/`
hierarchy. It is a structural-only edit; the verifier semantics are unchanged.

## Pinned circuit constants

The version pins inside `SP1VerifierGroth16.sol` provide an additional
integrity check — these MUST NOT change if the vendored binary is to remain
compatible with proofs generated by the matching prover binary:

| Constant | Value |
|---|---|
| `VERSION()` | `"v6.1.0"` |
| `VERIFIER_HASH()` | `0x4388a21c687fdd5f218d7e3d13190cac4c5355818d3605fd5fb811df468ee696` |
| `VK_ROOT()` | `0x002f850ee998974d6cc00e50cd0814b098c05bfade466d28573240d057f25352` |

The first 4 bytes of `VERIFIER_HASH()` (`0x4388a21c`) form the selector that
the canonical on-chain `SP1VerifierGateway` at `0x397A5f7f3dBd538f23DE225B51f532c34448dA9B`
(Base Sepolia) uses to route proofs to a v6.1.0 implementation. Producing a
verifier with the same selector but different constants is a circuit-version
mismatch and proofs will revert.

## Reproducing the vendor

```bash
mkdir /tmp/sp1-upstream && cd /tmp/sp1-upstream
curl -sLO https://raw.githubusercontent.com/succinctlabs/sp1-contracts/d3629729c3216eb51bd4859d027a8eb729399fa4/contracts/src/v6.1.0/Groth16Verifier.sol
curl -sLO https://raw.githubusercontent.com/succinctlabs/sp1-contracts/d3629729c3216eb51bd4859d027a8eb729399fa4/contracts/src/v6.1.0/SP1VerifierGroth16.sol
curl -sLO https://raw.githubusercontent.com/succinctlabs/sp1-contracts/d3629729c3216eb51bd4859d027a8eb729399fa4/contracts/src/ISP1Verifier.sol
shasum -a 256 *.sol
```

Expected SHA-256 (upstream):

```
01ab92508b9d56556ee247c7cb6b2ebd9c17835a4618896462b2a3a8708a50bf  Groth16Verifier.sol
9e918032a5aa799c1319b14b013154d6a40ca6e5f2267c9f540560abd7fd7689  ISP1Verifier.sol
48e1db5baca3b102242ebd88280b3689a088076688146cd0d98876f5dacb76d0  SP1VerifierGroth16.sol
```

The first two SHAs MUST match the local copies in this directory. The third
will differ by exactly the one-line `./` vs `../` import path flatten
documented above; diff `SP1VerifierGroth16.sol` against upstream and confirm
the only divergence is on line 4.

## Production deployment note

The CAW V2 deployment uses the canonical on-chain `SP1VerifierGateway` at
`0x397A5f7f3dBd538f23DE225B51f532c34448dA9B` (Base Sepolia), NOT this vendored
code. The vendored copy is exercised only by:

- `solidity/test/zk-real-verifier-test.js` (Truffle integration test against a
  locally-deployed verifier instance).
- Future deployments to chains where Succinct has not deployed an official
  gateway — in which case this vendored verifier becomes the canonical
  on-chain verifier and the provenance chain documented here becomes
  load-bearing.
